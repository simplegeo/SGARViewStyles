SG3DOverlayView

This view is in charge of allocating and maintaining the CAEAGLLayer that is to draw the AR environment.

Superclass: UIView
Declared In: SG3DOverlayView.h

Discussion

All touch events are sent through this view and they are handled by the SG3DOverlayViewDelegate . Although this view maintains the OpenGL layer, it is not in charge of setting it up or manipulating matrices. Once again, that job is delegated to SG3DOverlayViewDelegate.


Methods

-startAnimation

Adds a CADisplayLink to the current run loop which will call the drawView method for a given interval.

-stopAnimation

￼Removes the CADisplayLink from the current run loop and clears the OpenGL scene.
