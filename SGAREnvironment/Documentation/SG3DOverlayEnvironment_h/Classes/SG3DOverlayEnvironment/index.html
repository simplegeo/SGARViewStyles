SG3DOverlayEnvironment

This class is in charge of drawing all of the OpenGL components while producing the correct callback notifications for touch events generated by SG3DOverlayView.

Conforms to: SG3DOverlayViewDelegate, UIAccelerometerDelegate, CLLocationManagerDelegate
Superclass: NSObject
Declared In: SG3DOverlayEnvironment.h

Discussion

While the SG3DOverlayView is in charge of initializing the OpenGL environment, this class is responsible for "setting the stage". All operations on the projection and modelview matrices are done with drawView: and setupView: .

Within the drawView: method, the values from the accelerometer and the location manager are used to generate the proper matrices that will be applied to the modelview matrix. The accelerometer values are passed through a lowpass filter in order to generate the proper rotations matrices. The device, for the most part, will always be centered at the origin in the OpenGL environment. The floating SGAnnotationViews apply their own matrix transformations in order to be display in the 3D world properly.

Discussion

At this time, the camera is only asked to move when either the SG3DOverlayEnvrionment generates a pinch, pull or double tap control event. See SG3DOverlayViewDelegate . Also, enableWalking must be set to YES in order for the camera distance to be applied to the environment when the proper control signals are generated.
